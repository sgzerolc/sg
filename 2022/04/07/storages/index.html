time: 6.15-8.31-12.1

¶1 Overview

what are storage systems about?(concerns)

  1. Memory/storage hierarchy as a important aspect but not the focal point -> exploit locality -> power, bandwidth, management, …

    • Combining technologies(L1/L2 cache, DRAM, disk) to balance costs/benefits
    • similar to cache design for CPUs with differences

  2. Persistence

    want to do:

    • Store data for required time -> endure errors(machine failures, human, software)
    • find data-> data organization, consistency, management issues

¶compositions

  1. Storage systems: sw(file systems, device drivers, firmware) and hw(servers, devices, controllers, interconnect) that collectively provide application writers with an infrastructure that retains persistent data and provides it on request

  2. the concerned system: database, search engines, data analytics, social media, AI/ML, streaming process, …

  3. Viewpoint: defined by collection of layers and interfaces

    • Logical: Program <-> File System <-> Controller Firmware

    • Physical: Client<-> Server <-> I/O Controller <-> Device ASIC

  4. storage system interfaces

    • Naming and address mapping (e.g., relocation)
    • Caching and request transformations (e.g., combining)

¶goal

end-to-end understanding of storage systems:

  • key applications and technology trends
  • major components and fitting them together
  • storage management
  1. SSD and disk operation, firmware, and disk arrays
  2. file systems and database back-ends
  3. distributed file systems and parallel file systems
  4. virtualization and cluster storage
  5. storage management, data protection, disaster recovery
  6. alternate technologies, trends, industry viewpoints
