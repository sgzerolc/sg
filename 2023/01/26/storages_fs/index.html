¶Organizaing raw data into files

Problem 1: mapping file sys structure and data to storage locations(logical block numbers)

Basic fs structures: hierarchical structure has directories, files and symbolic links.

1.0 Directory -> Inode-> Data Block(LBAs)

1.1 More layers: DIdp -> Idp -> Dp -> Data Block, Inode has Direct pointer, Indirect pointer, Doubly-indirect pointer

Problem 2: tracking allocated/free space

  1. track free space: available LBNs tracking
  2. Allocate a free block when one is needed

ex: zonefs. It exposes each zone of a zoned block device as a file[1]. Superblock is written at LBA 0 which contains the implementations of zonefs. No complex in-disk metadata is involved.

organizations: /dev/nullb0 -> /mnt, conventional zones -> /mnt/conv/0~C, sequential zones -> /mnt/seq/0~S
